# 2️⃣ Backend Request Flow (How Every API Call Works)

Frontend Request
↓
app.js (Express receives request)
↓
auth.middleware.js (Validates JWT token)
↓
role.middleware.js (Checks role permission if required)
↓
routes/*.routes.js (Decides which controller to call)
↓
controllers/*.controller.js (Handles req & res)
↓
services/*.service.js (Business logic + DB query)
↓
MongoDB (Data read/write)
↓
Response sent back to frontend

---

# 3️⃣ Responsibilities of Each Layer

## 🔹 server.js

* Connects to MongoDB
* Starts Express server
* Reads PORT from .env

---

## 🔹 app.js

* Creates express app
* Adds middleware:

  * express.json()
  * cors()
* Registers routes:

  * /api/auth
  * /api/leads
  * /api/users
  * /api/reports
* Global error handler

---

## 🔹 config/db.js

* Connects to MongoDB using MONGO_URI
* Handles connection success/failure

---

## 🔹 routes/

Only defines:

* URL
* HTTP method
* Middleware
* Controller

No business logic here.

Example (lead.routes.js):

* GET /api/leads
* POST /api/leads
* GET /api/leads/:id
* PATCH /api/leads/:id
* DELETE /api/leads/:id (admin only)

---

## 🔹 controllers/

Handles:

* Reading req.body
* Reading req.params
* Reading req.query
* Reading req.user
* Sending response
* Error handling (try/catch)

NO database logic here.

---

## 🔹 services/

This is the core logic layer.

Responsibilities:

* Build MongoDB query
* Apply role-based filtering
* Execute Mongoose queries
* Return result

All business logic belongs here.

---

## 🔹 middleware/

### auth.middleware.js

* Reads Authorization header
* Verifies JWT token
* Attaches user to req.user
* Returns 401 if invalid

### role.middleware.js

* Checks if user.role is allowed
* Returns 403 if not allowed

---

## 🔹 models/

### User.js

Fields:

* name
* email (unique)
* password (hashed)
* role (admin | sales)
* timestamps

---

### Lead.js

Contains:

* srNo
* assignedTo (ref User)
* firmName
* personName
* mobileNo
* visitDate
* callingDate
* followUpDate
* bankName
* sanction
* sanctionDate
* amount
* projectType
* projectStatus
* timestamps

# 5️⃣ Role-Based Data Scoping (Important Logic)

Inside lead.service.js:

### getLeads(user)

If user.role === "admin"
→ Query = {}

If user.role === "sales"
→ Query = { assignedTo: user._id }

Return Lead.find(query)

---

### updateLead(leadId, data, user)

If admin
→ update by ID

If sales
→ update only if:
{ _id: leadId, assignedTo: user._id }

If not found
→ Throw error "Access denied"

---

### createLead(data, user)

Always:
data.assignedTo = user._id

Even if admin creates lead.

# 8️⃣ Error Handling Pattern

Controller:

```
try {
  const data = await serviceFunction()
  res.status(200).json(data)
} catch (error) {
  next(error)
}
```

Global Error Handler in app.js:

```
app.use((err, req, res, next) => {
  res.status(err.status || 500).json({
    message: err.message
  })
})
```
